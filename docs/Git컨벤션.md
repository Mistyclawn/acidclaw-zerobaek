# Git 커밋 메시지 컨벤션 (ZeroBaek)

본 프로젝트는 AcidClaw(오토 에이전트) 및 개발자(MistyClawn)가 일관된 형태로 커밋 히스토리를 관리하기 위해 아래의 컨벤션을 엄격히 따릅니다.

## 기본 구조
```text
<타입>: <제목>

<본문 (선택사항, 상세한 설명이 필요할 경우)>
```

## 타입 (Type) 목록
- **feat**: 새로운 기능 추가
- **fix**: 버그 수정
- **docs**: 문서 수정 (README, md 파일 등)
- **style**: 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우 (CSS 스타일링 아님)
- **refactor**: 코드 리팩터링
- **test**: 테스트 코드, 리팩터링 테스트 코드 추가
- **chore**: 빌드 업무 수정, 패키지 매니저 수정, 그 외 자잘한 작업

## 예시
**좋은 예:**
- `feat: LMB/RMB 양발 스텝 이벤트 리스너 추가`
- `docs: 크론잡 명세에 보고 채널(#zerobaek-오토개발) 정보 동기화`
- `fix: 윈도우 리사이즈 시 Canvas 비율 깨지는 현상 수정`

**나쁜 예 (사용 금지):**
- `코드 짰음`
- `Update README.md`
- `오류 고침`

**[주의]** 크론잡 오토 개발 시에도 이 컨벤션 룰을 반드시 상기하여 일관된 로그를 남겨야 합니다.
